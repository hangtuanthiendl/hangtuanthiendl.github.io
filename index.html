Abnormal log detection & Self-healing System

Logs and metrics of requests will be collected by a validated service. This service then stores the data and publishes them to Kafka. The objective of this project is to group the data in a window of time and detect abnormal data such as DDOS attack, Internal Exception (500), Bottle Neck, ...

What I did:

- Use Scylla (based on Cassandra Database) and Postgres SQL to store raw logs and processing logs respectively which are converted into bit numbers.

- Use PyFlink to extract the data from Kafka, group them as a window, transform and load them back to Postgres SQL.

- Use PySpark to detect the abnormal data with LSTM model before loading them back to Scylla.

Other tech stack: Faust, Faust Streaming, Websocket, HTTP2
